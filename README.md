# 🚑 AGV 활용 자택 응급 조치 프로토콜 (Home Emergency Response Protocol via AGV & Wi-Fi DensePose)

본 프로젝트는 **Wi-Fi 센싱 및 Linux 기반 DensePose** 기술을 활용하여 벽 뒤에 있는 사람의 쓰러짐 사고를 감지하고, 이와 연동된 **Jetson Nano 기반 AGV(자율주행 무인 운반차)**를 출동시켜 환자의 상태 파악 및 응급 조치를 수행하는 스마트 홈 헬스케어 시스템입니다.

---

## 🛠️ 시스템 아키텍처 및 역할 분담 (System Architecture)

시스템은 **메인 감지 서버(Linux + STM32/ESP32)**와 호출을 받아 이동하는 **이동형 AGV(Jetson Nano)**로 이원화되어 제어됩니다.


```
[벽 뒤의 환자]
│ (Wi-Fi CSI 신호 변화 발생)
[ESP32 / Wi-Fi Sensing Node] ──> 데이터 수집 후 메인 PC로 전달
│
[Linux 메인 서버] ──> Wi-Fi DensePose 모델 구동 (쓰러짐 감지 및 격자 위치 추정)
│ (Wi-Fi / Call 명령 전송)
[Jetson Nano (AGV 메인)] ──> SLAM 기반 지도 확인 및 최단 경로 자율주행 출동
│ (모터/센서 하위 제어)
[STM32 (AGV 하위)] ──> 모터 엔코더 및 실시간 하드웨어 제어
```

---

## 📌 핵심 마일스톤 (Milestones)

### 1단계: 와이파이 센싱 및 Linux DensePose를 통한 쓰러짐/위치 감지
Wi-Fi 신호가 벽을 통과할 때 발생하는 회절과 반사 특성인 **CSI(Channel State Information)**를 이용합니다.
* **사람의 쓰러짐 감지 (Wi-Fi DensePose)**
  * **구현**: 하드웨어 노드(ESP32/STM32)를 통해 실내의 Wi-Fi CSI 데이터를 수집하여 **Linux 메인 서버**로 전송합니다.
  * **모델 구동**: Linux 서버에서 **Wi-Fi DensePose** 모델을 구동하여 3D 신체 표면 및 자세를 추정하고, 사람이 쓰러진 상태를 인공지능으로 정확하게 판별합니다.
* **쓰러진 사람의 위치 체크**
  * 실내에 배치된 다수의 노드 신호 데이터를 기반으로 삼각측량 및 DensePose 공간 매핑을 통해 환자가 쓰러진 위치를 격자(Grid) 좌표 형태로 추정합니다.

### 2단계: AGV 호출 및 자율 이동 (Jetson Nano 메인 제어)
Linux 메인 서버로부터 비상 호출(Call)을 받은 AGV가 환자의 위치로 정확하게 찾아가는 단계입니다.
* **집안 스캐닝 (SLAM / Map 만들기)**
  * AGV의 메인 프로세서인 **Jetson Nano**를 중심으로 맵핑을 수행합니다. 
  * AGV에 탑재된 LiDAR(라이다) 센서 데이터와 **STM32(하위 제어기)**에서 연산된 모터 엔코더 값(Odometry)을 Jetson Nano가 취합하여 집안의 2D/3D 지도를 그리는 SLAM을 구현합니다.
* **호출된 AGV가 응급환자에게 이동 (Navigation)**
  * 메인 서버로부터 환자의 격자 좌표 정보를 수신하면, Jetson Nano에서 ROS(Robot Operating System) 기반의 Navigation 스택을 구동합니다.
  * 생성된 지도 위에서 실시간 장애물을 회피하며 최단 거리로 이동(A* 알고리즘 등)하며, 하위 주행 제어는 STM32에 명령을 내려 모터를 구동합니다.

### 3단계: 응급환자 상태 체크 및 프로토콜 실행
환자에게 도착한 AGV가 현장 상태를 파악하고 실제 응급 상황 여부를 최종 판단합니다.
* **쓰러진 사람의 상태 체크**
  * **비접촉 센싱**: Jetson Nano에 연결된 카메라(OpenCV 기반 AI 객체 검출로 누워있는 자세 분석) 또는 열화상 카메라를 통해 환자의 실시간 상태와 체온 변화를 확인합니다.
  * **안내 및 반응 확인**: AGV에 장착된 스피커를 통해 "괜찮으십니까? 괜찮으시면 버튼을 누르거나 대답하세요" 등의 안내 방송을 송출하고 환자의 반응을 모니터링합니다.
* **최종 프로토콜 실행**
  * 일정 시간 동안 환자의 반응이 없거나 위급 상황으로 판단될 경우, 메인 서버 및 AGV의 통신 기능을 통해 외부 기관(119) 및 보호자 스마트폰으로 비상 알림, 현재 위치 좌표, 환자 상태 스냅샷을 즉시 전송합니다.

## 💡 주요 고려사항 (Technical Tips)
* **Linux - Jetson Nano 간의 Call 통신**: 메인 서버가 환자를 감지하고 Jetson Nano로 명령을 내릴 때, ROS2의 Topic/Service 통신이나 MQTT, Socket 통신 프로토콜을 활용하여 끊김 없고 지연 없는 구조 신호 전달 체계를 구축해야 합니다.
* **CSI 데이터 노이즈 필터링**: Wi-Fi CSI 데이터는 주변 환경(반려동물, 문 열림 등)에 매우 민감하므로 DensePose 모델 입력 전 초기 데이터 필터링(Bandpass Filter 등) 공정이 중요합니다.
